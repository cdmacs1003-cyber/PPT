# 단계별 작업 프롬프트 모음

이 문서는 AI 자동화 컨설팅 제안서 PPT 프로젝트를 진행할 때 Codex에게 복사해서 입력하기 좋은 프롬프트 모음입니다.

복잡한 명령보다 바로 이해되는 말로 구성했습니다.

## 1단계: 개발 전 환경 준비

프로젝트 폴더와 기본 문서를 준비할 때 사용합니다.

```text
목표:
이 프로젝트를 AI 자동화 컨설팅 제안서 PPT 제작용 작업 폴더로 준비해줘.

성공 기준:
- AGENTS.md 생성
- brand/DESIGN.md 생성
- context/business-context.md 생성
- context/deck-brief.md 생성
- assets/와 output/ 폴더 생성
- 각 파일에는 초안 내용이 들어가야 함

제약:
- 아직 PPT 파일은 만들지 말 것
- 특정 브랜드의 공식 디자인 시스템이라고 쓰지 말 것
- 초보자가 읽어도 이해되는 쉬운 말로 작성할 것

출력:
- 만든 파일 목록
- 각 파일의 역할 한 줄 설명
- 다음에 내가 확인해야 할 것
```

## 2단계: 프로젝트 구조 점검

작업을 시작하기 전에 현재 폴더가 잘 준비됐는지 확인할 때 사용합니다.

```text
현재 프로젝트 폴더를 점검해줘.

확인할 것:
- 필요한 파일과 폴더가 모두 있는지
- 비어 있거나 내용이 부족한 문서가 있는지
- PPT 제작 전에 더 채워야 할 정보가 무엇인지

출력:
- 준비 완료된 항목
- 부족한 항목
- 다음에 해야 할 일 3개

주의:
- 파일을 수정하지 말고 먼저 점검 결과만 알려줘.
```

## 3단계: 고객/사업 정보 정리

제안서의 방향을 잡기 위해 고객 정보를 정리할 때 사용합니다.

```text
AI 자동화 컨설팅 제안서에 들어갈 사업 정보를 정리해줘.

내가 제공할 정보:
- 고객 회사명:
- 업종:
- 가장 불편한 업무:
- 제안서 발표 대상:
- 원하는 결과:

작업:
- context/business-context.md를 초보자도 이해하기 쉬운 말로 정리
- 확정된 정보와 아직 모르는 정보를 나눠서 작성
- 과장된 수치나 확인되지 않은 성과는 쓰지 말 것

출력:
- 수정한 내용 요약
- 추가로 확인해야 할 질문
```

## 4단계: 디자인 방향 정리

PPT의 분위기와 디자인 기준을 잡을 때 사용합니다.

```text
AI 자동화 컨설팅 제안서의 디자인 방향을 정리해줘.

원하는 느낌:
- 전문적
- 깔끔함
- 신뢰감
- 너무 화려하지 않음

작업:
- brand/DESIGN.md를 업데이트
- 색상, 글꼴, 슬라이드 구성, 이미지 사용 방향을 정리
- 특정 브랜드의 공식 디자인 시스템이라고 쓰지 말 것

출력:
- 디자인 방향 요약
- PPT 만들기 전에 필요한 자료 목록
```

## 5단계: PPT 구성안 만들기

실제 PPT를 만들기 전에 슬라이드 순서를 정할 때 사용합니다.

```text
AI 자동화 컨설팅 제안서 PPT의 구성안을 만들어줘.

조건:
- 아직 PPT 파일은 만들지 말 것
- 먼저 context/deck-brief.md에 슬라이드 구성 초안을 정리할 것
- 초보자가 봐도 발표 흐름을 이해할 수 있게 작성할 것

포함할 내용:
- 표지
- 고객 문제
- AI 자동화가 필요한 이유
- 적용 가능한 업무 예시
- 컨설팅 진행 방식
- 기대 효과
- 일정과 산출물
- 다음 단계

출력:
- 예상 슬라이드 목록
- 각 슬라이드의 핵심 메시지
- 더 필요한 고객 정보
```

## 6단계: 자료 수집 준비

PPT에 들어갈 이미지, 로고, 참고자료를 정리할 때 사용합니다.

```text
PPT 제작에 필요한 자료 목록을 정리해줘.

작업:
- assets/ 폴더에 넣어야 할 자료 목록 작성
- 로고, 이미지, 참고 문서, 기존 제안서, 고객 자료를 구분
- 내가 어떤 파일을 준비해야 하는지 쉽게 알려줘

출력:
- 필요한 자료 체크리스트
- 자료가 없을 때 대체할 방법
- 다음 작업 순서

주의:
- 아직 PPT 파일은 만들지 말 것
```

## 7단계: PPT 초안 제작

준비 문서가 충분히 채워진 뒤 실제 PPT 초안을 만들 때 사용합니다.

```text
준비된 문서를 바탕으로 AI 자동화 컨설팅 제안서 PPT 초안을 만들어줘.

사용할 자료:
- AGENTS.md
- brand/DESIGN.md
- context/business-context.md
- context/deck-brief.md
- assets/ 폴더 자료

작업:
- output/ 폴더에 PPT 초안 생성
- 각 슬라이드는 한 가지 메시지만 담기
- 어려운 용어는 쉬운 말로 바꾸기
- 확인되지 않은 숫자나 성과는 넣지 않기

출력:
- 만든 PPT 파일명
- 슬라이드 목록
- 내가 검토해야 할 부분
```

## 8단계: 검토와 수정

PPT 초안을 만든 뒤 품질을 확인할 때 사용합니다.

```text
만든 PPT 초안을 검토해줘.

확인할 것:
- 발표 흐름이 자연스러운지
- 고객 문제와 해결책이 잘 연결되는지
- 너무 어려운 말이 있는지
- 디자인이 일관적인지
- 확인되지 않은 주장이나 수치가 있는지

출력:
- 잘 된 점
- 수정이 필요한 점
- 바로 고치면 좋은 우선순위 5개

주의:
- 먼저 검토 결과를 알려주고, 수정은 내가 요청하면 진행해줘.
```

## 9단계: 최종본 정리

발표 전 최종 파일을 정리할 때 사용합니다.

```text
PPT 최종본을 만들기 전에 파일과 내용을 정리해줘.

작업:
- output/ 폴더의 결과물 확인
- 파일명이 알아보기 쉬운지 확인
- 발표 전에 빠진 정보가 없는지 확인
- 최종 검토 체크리스트 작성

출력:
- 최종본 후보 파일
- 발표 전 확인할 것
- 수정하면 좋은 마지막 항목
```

## 10단계: GitHub 저장

작업 내용을 GitHub에 올리기 전에 사용합니다.

```text
현재 변경사항을 GitHub에 올려도 되는지 확인해줘.

작업:
- 변경된 파일 목록 확인
- 민감한 정보가 들어갔는지 확인
- 커밋 메시지 추천
- GitHub에 올리는 명령어를 한 줄씩 안내

출력:
- 올려도 되는 파일
- 조심해야 할 파일
- 추천 커밋 메시지
- 실행할 명령어

주의:
- 내가 허락하기 전에는 git commit이나 git push를 실행하지 말 것
```

## 11단계: 배포 또는 공유 준비

완성된 PPT를 공유하기 전에 사용합니다.

```text
완성된 PPT를 공유하기 전에 배포 준비를 도와줘.

확인할 것:
- 최종 PPT 파일 위치
- 공유해도 되는 내용인지
- 고객명, 로고, 민감한 정보가 맞게 들어갔는지
- PDF 변환이 필요한지
- 공유용 파일명은 적절한지

출력:
- 공유 전 체크리스트
- 추천 파일명
- 공유할 때 같이 보낼 짧은 안내문

주의:
- 외부 업로드나 배포는 내가 직접 확인한 뒤 진행할 것
```
