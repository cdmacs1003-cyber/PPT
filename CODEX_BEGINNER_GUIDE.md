# Codex 초보자 설정/사용 가이드

이 문서는 코딩을 처음 시작하는 사람도 Codex를 안전하게 쓰기 위한 약속과 사용법입니다.

## 현재 추천 설정

- 모델: `gpt-5.5` 유지
- 추론 강도: `high` 유지
- 브라우저 플러그인: 사용 가능
- 작업 방식: 먼저 설명, 그 다음 수정, 마지막에 확인

초보자에게는 빠른 실행보다 안전한 확인이 더 중요합니다. Codex에게 작업을 맡길 때는 "먼저 쉽게 설명해줘"라고 말하면 좋습니다.

## Codex에게 이렇게 말하기

코드를 고치고 싶을 때:

```text
이 기능을 만들고 싶은데, 초보자도 이해하게 먼저 계획을 설명하고 수정해줘.
```

에러가 났을 때:

```text
이 에러를 초딩 버전으로 설명하고, 고치는 명령어를 한 줄씩 알려줘.
```

GitHub에 올릴 때:

```text
현재 변경사항 확인하고, 커밋 메시지 추천한 뒤 GitHub에 올리는 순서를 알려줘.
```

프로젝트 구조가 궁금할 때:

```text
이 프로젝트 구조를 초보자 눈높이로 설명해줘.
```

## 안전 습관

- 작업 전에는 `현재 상태 확인해줘`라고 말합니다.
- 수정 후에는 `뭐가 바뀌었는지 쉬운 말로 요약해줘`라고 말합니다.
- GitHub에 올리기 전에는 `올려도 되는 변경인지 확인해줘`라고 말합니다.
- 삭제가 필요하면 `삭제해도 안전한지 먼저 설명해줘`라고 말합니다.

## 위험한 작업 전에 확인할 것

아래 작업은 바로 실행하지 말고 먼저 이유를 들어보는 것이 좋습니다.

- 파일이나 폴더 삭제
- GitHub 업로드
- 프로그램 설치
- 배포
- Git 기록 변경

## 기본 약속

Codex를 초보자 모드로 사용할 때는 아래 방식이 좋습니다.

1. 어려운 용어는 쉬운 말로 풀어서 설명합니다.
2. 명령어는 한 줄씩 안내합니다.
3. 큰 변경 전에는 무엇을 바꾸는지 먼저 설명합니다.
4. 작업 후에는 바뀐 내용을 짧게 요약합니다.
5. Git으로 되돌릴 수 있게 중간중간 저장합니다.

## 자주 쓰는 Git 명령어

변경된 파일 확인:

```powershell
git status
```

변경사항 저장 준비:

```powershell
git add .
```

내 컴퓨터 Git에 저장:

```powershell
git commit -m "Update project"
```

GitHub에 올리기:

```powershell
git push
```
